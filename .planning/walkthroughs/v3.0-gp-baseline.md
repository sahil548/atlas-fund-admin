During v2.1 the `/api/entities` endpoint response shape appears to have changed (likely paginated or object-wrapped), so `entities` is now an object rather than an array; `(entities || []).map` fails because truthy-but-non-array objects skip the `|| []` fallback. **Fix scope:** either (a) normalize `/api/entities` and `/api/investors` endpoints to always return an array at the top level, or (b) update this form (and every similar consumer) to explicitly unwrap `.items` / `.data` safely. Option (a) is the conservative path; option (b) is the one that unblocks v3.0 Phase 25 (pagination) without another forced response-shape migration.
