**IMPORTANT:** The tree view needs entities with their `childEntities` included. The flat API response does NOT include children. To avoid N+1 queries, the entities page will pass a locally-constructed tree (build from `parentEntityId` field on the flat entity list — all entities are already loaded, just need to nest them client-side).

Client-side tree building utility function (inside tree view file):
```typescript
function buildTree(entities: any[]) {
  const map = new Map(entities.map(e => [e.id, { ...e, childEntities: [] }]));
  const roots: any[] = [];
  for (const entity of map.values()) {
    if (entity.parentEntityId && map.has(entity.parentEntityId)) {
      map.get(entity.parentEntityId)!.childEntities.push(entity);
    } else {
      roots.push(entity);
    }
  }
  return roots;
}
```
