    ```typescript

      const perms = await getEffectivePermissions(authUser.id);
    ```

    For POST/PUT/PATCH/DELETE handlers (write operations):
    ```typescript
    if (authUser.role === "GP_TEAM") {
      const perms = await getEffectivePermissions(authUser.id);
      if (!checkPermission(perms, "{AREA}", "full")) return forbidden();
    }
    ```

    The {AREA} mapping:
    - deals/route.ts and deals/[id]/route.ts: `"deals"`
    - entities/route.ts and entities/[id]/route.ts: `"entities"`
    - capital-calls/* routes: `"capital_activity"`
    - distributions/* routes: `"capital_activity"`

    For `entities/[id]/route.ts` GET/PUT (single): If `authUser.role === "SERVICE_PROVIDER"` and `!authUser.entityAccess.includes(id)`, return forbidden().

    For `capital-calls/route.ts` GET (list): If `authUser.role === "SERVICE_PROVIDER"`, add `entity: { id: { in: authUser.entityAccess } }` to the where clause.

    For `capital-calls/[id]/route.ts` GET (single): After fetching the capital call, if `authUser.role === "SERVICE_PROVIDER"` and the capital call's entityId is not in `authUser.entityAccess`, return forbidden().

    Same pattern for `distributions/route.ts` and `distributions/[id]/route.ts`.

    For line-items routes: The parent capital-call/distribution is already entity-scoped, so entity-scope on line items is inherited. Still add the GP_TEAM permission gate for `"capital_activity"`.
