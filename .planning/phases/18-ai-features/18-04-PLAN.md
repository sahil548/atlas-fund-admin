    **1. Create `src/app/api/ai/suggest-tasks/route.ts`:**

    POST endpoint that generates context-aware task suggestions for a specific deal/asset/entity.

    Request schema:
    ```typescript
    const SuggestTasksSchema = z.object({
      firmId: z.string().min(1),
      contextType: z.enum(["DEAL", "ASSET", "ENTITY"]),
      contextId: z.string().min(1),
    });
    ```

    Implementation:
    - Use `getAuthUser()` for userId
    - Use `createUserAIClient(userId, firmId)` — return "No API key" error if null
    - Fetch context data based on contextType:
      - DEAL: `prisma.deal.findUnique({ where: { id: contextId }, include: { activities: { orderBy: { createdAt: "desc" }, take: 5 }, tasks: { take: 10 }, documents: { take: 5 } } })` — get deal name, stage, asset class, recent activity, existing tasks, documents
      - ASSET: `prisma.asset.findUnique(...)` — get asset name, class, status, fair value, recent tasks
      - ENTITY: `prisma.entity.findUnique(...)` — get entity name, type, status, related assets
    - Build prompt (from RESEARCH.md Pattern 7):
      ```
      Given this {contextType} context, suggest 3-5 specific next tasks the GP should do.
      Return as JSON: { "tasks": [{ "title": "...", "priority": "HIGH" | "MEDIUM" | "LOW", "rationale": "..." }] }

      {contextType}: {name}, Stage/Status: {stage/status}, Type/Class: {type/assetClass}
      Recent activity: {last 3-5 activities}
      Existing tasks: {current tasks, to avoid duplicates}
      Missing fields: {null/empty important fields}
      ```
    - Parse response with `jsonrepair` (import from "jsonrepair")
    - Return `{ tasks: Array<{ title: string; priority: string; rationale: string }> }`
    - Apply rate limiting

    **2. Create `src/app/api/ai/draft-lp-update/route.ts`:**

    POST endpoint that generates an LP quarterly update draft for a specific fund entity.

    Request schema:
    ```typescript
    const DraftLPUpdateSchema = z.object({
      firmId: z.string().min(1),
      entityId: z.string().min(1),
    });
    ```

    Implementation:
    - Use `getAuthUser()` for userId
    - Use `createUserAIClient(userId, firmId)` — return "No API key" error if null
    - Gather fund data (cap data sent to LLM per RESEARCH.md Pitfall 5):
      ```typescript
      const entity = await prisma.entity.findUnique({
        where: { id: entityId },
        include: {
          assets: { take: 10, orderBy: { fairValue: "desc" }, select: { name: true, assetClass: true, fairValue: true, costBasis: true, status: true } },
          capitalCalls: { take: 5, orderBy: { createdAt: "desc" }, select: { totalAmount: true, status: true, createdAt: true } },
          distributions: { take: 5, orderBy: { createdAt: "desc" }, select: { totalAmount: true, status: true, createdAt: true } },
        }
      });
      ```
    - Format numbers using `fmt()` from `@/lib/utils` before sending to prompt (keeps token count low)
    - Build prompt (from RESEARCH.md Pattern 8):
      ```
      You are drafting a quarterly LP update for {entity.name}.

      Fund Data:
      - Type: {entity.type}
      - Total Commitments: {fmt(entity.totalCommitments)}
      - Current NAV: {fmt(totalFairValue)}
      - Assets: {list with name, class, fair value, cost basis}
      - Recent Capital Calls: {amounts, dates, status}
      - Recent Distributions: {amounts, dates, status}

      Write a professional LP quarterly update. Format:
      1. Executive Summary (2-3 sentences)
      2. Portfolio Highlights
      3. Capital Activity
      4. Outlook

      Be factual, use the data provided, professional tone.
      ```
    - Return `{ draft: string, entityName: string }` — the draft as plain text/markdown
    - Apply rate limiting

    **Key patterns:**
    - Both endpoints use `createUserAIClient(userId, firmId)` (Phase 12 pattern, user key -> tenant key -> none)
    - Both use `jsonrepair` for parsing AI JSON responses
    - Both cap data sent to LLM to prevent token limit issues
    - Both use `parseBody()` + Zod for request validation
    - Both use `getAuthUser()` for auth

    **1. Task suggestion rendering in command bar and side panel:**

    When the AI response includes task suggestions (either from the /api/ai/execute response for SUGGEST_TASKS actionType, or when the GP asks "what should I do next on this deal" and the AI search recognizes it), render task suggestion cards:

    Add a detection flow: When the AI query response mentions task suggestions or the user's intent maps to SUGGEST_TASKS, call `/api/ai/suggest-tasks` with the current page context:
    ```typescript
    // In the command bar's AI response handling:
    // If pageContext exists and query matches task suggestion pattern
    const isSuggestQuery = /what should (i|we) do|next (steps?|tasks?|actions?)|suggest.*tasks?/i.test(query);
    if (isSuggestQuery && pageContext?.entityId) {
      const contextTypeMap: Record<string, string> = { deal: "DEAL", asset: "ASSET", entity: "ENTITY" };
      const contextType = contextTypeMap[pageContext.pageType];
      if (contextType) {
        const res = await fetch("/api/ai/suggest-tasks", {
          method: "POST",
          headers: { "Content-Type": "application/json" },
          body: JSON.stringify({ firmId, contextType, contextId: pageContext.entityId }),
        }).then(r => r.json());
        // Render task cards
      }
    }
    ```

    Each task suggestion card renders as:
    ```
    [Task title]                          [HIGH] [+ Add]
    [rationale in small gray text]
    ```
    - The "Add" button calls `POST /api/tasks` with `{ title, priority, contextType, contextId: pageContext.entityId, dealId/assetId/entityId }` using the `useFirm()` firmId
    - After adding: show toast "Task created" and visually mark that suggestion as "added" (gray out the Add button, show checkmark)
    - Use `const toast = useToast()` (no destructuring per coding-patterns.md)

    Render these cards in both the command bar dropdown and side panel conversation thread.

    ```typescript
    const isLPUpdate = /draft.*lp.*update|lp.*quarterly|investor.*update|lp.*report/i.test(query);
    ```

    ```typescript
    const res = await fetch("/api/ai/draft-lp-update", {
    ```
