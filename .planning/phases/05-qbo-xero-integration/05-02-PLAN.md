All routes use `{ params }: { params: Promise<{ connectionId: string }> }` and `await params` (Next.js 16 pattern). All routes authenticate with `getAuthUser()` and verify the connection belongs to the user's firm via entity->firmId check.

**GET /api/accounting/connections/[connectionId]/chart-of-accounts/route.ts:**

1. Load AccountingConnection with entity (for firm check).
2. Call `getValidTokens(connectionId)` — if null, return 401 "OAuth tokens expired, please reconnect".
3. Call `qboProvider.fetchChartOfAccounts(tokens)`.
4. For each account, call `detectAtlasBucket(account)` to get suggested mapping.
5. Load existing `AccountMapping` records for this connection.
6. Return response shape:
```json
{
  "accounts": [
    {
      "accountId": "1",
      "accountName": "Checking",
      "accountType": "Bank",
      "classification": "Asset",
      "currentBalance": 50000,
      "suggestedBucket": "CASH",
      "currentMapping": "CASH" | null,
      "isMapped": true | false
    }
  ],
  "unmappedCount": 3,
  "totalAccounts": 25
}
```

**GET + POST + PATCH /api/accounting/connections/[connectionId]/mappings/route.ts:**

GET: Return all AccountMapping records for this connection, joined with provider account info.

POST: Bulk create/update mappings. Accept body:
```json
{
  "mappings": [
    { "providerAccountId": "1", "providerAccountName": "Checking", "atlasAccountType": "CASH", "isAutoDetected": true },
    { "providerAccountId": "2", "providerAccountName": "Investment Portfolio", "atlasAccountType": "INVESTMENTS_AT_COST", "isAutoDetected": false }
  ]
}
```
Use a Zod schema `BulkAccountMappingSchema = z.object({ mappings: z.array(CreateAccountMappingSchema.omit({ connectionId: true })) })`.
For each mapping: upsert by (connectionId, providerAccountId). After all mappings saved, update `AccountingConnection.chartOfAccountsMapped = true` if all balance-sheet accounts are mapped.

Also implement "suggest from last mapped entity" logic per user decision: Accept optional `?suggestFrom=connectionId` query param on the chart-of-accounts GET. If provided, load that connection's mappings and use them as suggestions instead of auto-detect for matching account names.

PATCH: Update single mapping's `atlasAccountType`. Accept `{ mappingId, atlasAccountType }`.

**POST /api/accounting/connections/[connectionId]/sync/route.ts:**

This is the "Sync Now" button handler.

1. Load connection, verify firm.
2. Get valid tokens (return error if expired).
3. Set `syncStatus: SYNCING` on connection.
4. Accept optional `?asOfDate=YYYY-MM-DD` query param. Default: last day of previous month.
5. Call `qboProvider.fetchTrialBalance(tokens, asOfDate)`.
6. Upsert `TrialBalanceSnapshot`:
   - `connectionId`, `periodDate` (as-of date), `accountData` (the entries array as JSON), `totalDebits`, `totalCredits`, `isBalanced` (Math.abs(totalDebits - totalCredits) < 0.01).
7. Update connection: `syncStatus: CONNECTED`, `lastSyncAt: new Date()`, `lastSyncError: null`.
8. If isBalanced is false, set `unreconciledItems` to the count of accounts where debit != credit unexpectedly.
9. On any error: set `syncStatus: ERROR`, `lastSyncError: error.message`, keep `lastSyncAt` unchanged (last successful sync).
10. Return the snapshot data with sync status.

**GET /api/accounting/connections/[connectionId]/trial-balance/route.ts:**

1. Accept optional `?periodDate=YYYY-MM-DD` to get specific snapshot.
2. If no periodDate, return most recent snapshot.
3. Also accept `?list=true` to return list of all available period dates (for period selector dropdown).
4. For single snapshot response, organize the accountData by Atlas bucket using the connection's AccountMapping records:
