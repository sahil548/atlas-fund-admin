**NAV endpoint GL branch (src/app/api/nav/[entityId]/route.ts):**

Modify the existing GET handler. The entity query already includes `accountingConnection: true`. Add to the include: `accountingConnection: { include: { accountMappings: true, trialBalanceSnapshots: { orderBy: { periodDate: "desc" }, take: 1 } } }`.

After loading the entity, determine the cost basis source:

```typescript
const connection = entity.accountingConnection;
const hasGLData = connection
  && connection.syncStatus !== "DISCONNECTED"
  && connection.chartOfAccountsMapped === true
  && connection.trialBalanceSnapshots?.length > 0;
```

If `hasGLData` is true, use GL-based cost basis:
1. Get the most recent `TrialBalanceSnapshot` (`connection.trialBalanceSnapshots[0]`).
2. Parse `accountData` (JSON array of trial balance entries).
3. Build a map: `providerAccountId -> atlasAccountType` from `connection.accountMappings`.
4. Sum trial balance entries by their mapped Atlas bucket:
   - `CASH`: sum of balances for accounts mapped to CASH
   - `INVESTMENTS_AT_COST`: sum of balances for accounts mapped to INVESTMENTS_AT_COST
   - `OTHER_ASSETS`: sum of balances for accounts mapped to OTHER_ASSETS
   - `LIABILITIES`: sum of absolute balances for accounts mapped to LIABILITIES (liabilities have credit balances, so use absolute value or handle sign convention)
   - `EQUITY_PARTNERS_CAPITAL`: sum for equity accounts (informational, not used in NAV calc directly)
5. Compute GL-based Layer 1:
   ```typescript
   const glCash = bucketTotals.CASH || 0;
   const glInvestments = bucketTotals.INVESTMENTS_AT_COST || 0;
   const glOtherAssets = bucketTotals.OTHER_ASSETS || 0;
   const glLiabilities = Math.abs(bucketTotals.LIABILITIES || 0);
   const totalAssets = glCash + glInvestments + glOtherAssets;
   const costBasisNAV = totalAssets - glLiabilities;
   ```
6. Replace `investmentsAtCost` with `glInvestments` for the fair value overlay calculation (allocationPercent still applies to fair value layer).

If `hasGLData` is false, keep the existing proxy-based calculation exactly as-is.

Add to the response:
```typescript
navSource: hasGLData ? "GL" : "PROXY",
glDataAsOf: hasGLData ? connection.trialBalanceSnapshots[0].periodDate : null,
lastSyncAt: connection?.lastSyncAt || null,
syncStatus: connection?.syncStatus || null,
```

The Layer 2 (fair value overlay) remains unchanged — it always uses Atlas asset valuations regardless of GL connection.

The NAV snapshot upsert continues as-is with the new values.

IMPORTANT: The existing proxy variables (`cashPercent`, `otherAssetsPercent`, `liabilitiesPercent`) and proxy calculation block must remain for the fallback case. Structure as an if/else with the GL branch first.
