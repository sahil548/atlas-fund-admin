```typescript
// Provider-agnostic interfaces — Xero will implement these same interfaces later

export interface OAuthTokens {
  accessToken: string;
  refreshToken: string;
  accessTokenExpiresAt: string;  // ISO date
  refreshTokenExpiresAt: string; // ISO date
  realmId: string;               // QBO company ID / Xero tenant ID
}

export interface ChartOfAccountsEntry {
  accountId: string;
  accountName: string;
  accountType: string;        // provider-native type (e.g., "Bank", "Accounts Receivable")
  accountSubType?: string;    // provider-native subtype
  currentBalance: number;
  classification: string;     // "Asset", "Liability", "Equity", "Revenue", "Expense"
  isActive: boolean;
}

export type AtlasAccountBucket =
  | "CASH"
  | "INVESTMENTS_AT_COST"
  | "OTHER_ASSETS"
  | "LIABILITIES"
  | "EQUITY_PARTNERS_CAPITAL";

export interface TrialBalanceEntry {
  accountId: string;
  accountName: string;
  accountType: string;
  debit: number;
  credit: number;
  balance: number;
}

export interface TrialBalanceResult {
  periodDate: string;         // ISO date — as-of date
  entries: TrialBalanceEntry[];
  totalDebits: number;
  totalCredits: number;
  isBalanced: boolean;
}

export interface AccountingProviderInterface {
  /** Get the OAuth authorization URL to redirect the user to */
  getAuthorizationUrl(entityId: string, state: string): string;

  /** Exchange authorization code for tokens */
  exchangeCodeForTokens(code: string, realmId: string): Promise<OAuthTokens>;

  /** Refresh expired access token using refresh token */
  refreshAccessToken(refreshToken: string): Promise<OAuthTokens>;

  /** Fetch chart of accounts from the provider */
  fetchChartOfAccounts(tokens: OAuthTokens): Promise<ChartOfAccountsEntry[]>;

  /** Fetch trial balance for a given period */
  fetchTrialBalance(tokens: OAuthTokens, asOfDate: string): Promise<TrialBalanceResult>;

  /** Revoke OAuth tokens (on disconnect) */
  revokeTokens(tokens: OAuthTokens): Promise<void>;
}

/** Auto-detect which Atlas bucket a provider account maps to based on account type/classification */
export function detectAtlasBucket(entry: ChartOfAccountsEntry): AtlasAccountBucket | null {
  const type = entry.accountType.toLowerCase();
  const subType = (entry.accountSubType || "").toLowerCase();
  const name = entry.accountName.toLowerCase();
  const classification = entry.classification.toLowerCase();

  // Cash detection
  if (type === "bank" || subType.includes("cash") || subType.includes("checking") || subType.includes("savings") || subType.includes("money market")) {
    return "CASH";
  }

  // Investments at cost — look for investment-related accounts
  if (name.includes("investment") || name.includes("portfolio") || subType.includes("investment") || (classification === "asset" && (name.includes("equity") || name.includes("loan") || name.includes("note receivable")))) {
    return "INVESTMENTS_AT_COST";
  }

  // Liabilities
  if (classification === "liability") {
    return "LIABILITIES";
  }

  // Equity / Partners' Capital
  if (classification === "equity" || type.includes("equity") || name.includes("partner") || name.includes("capital")) {
    return "EQUITY_PARTNERS_CAPITAL";
  }

  // Other assets (catch-all for remaining asset accounts)
  if (classification === "asset") {
    return "OTHER_ASSETS";
  }

  // Revenue/Expense accounts don't map to any bucket (balance sheet only)
  return null;
}
```

**QBO Provider in src/lib/accounting/qbo-provider.ts:**

Implement `AccountingProviderInterface` for QBO. Key details:
- Authorization URL: `https://appcenter.intuit.com/connect/oauth2` with `scope=com.intuit.quickbooks.accounting` and `response_type=code`
- Token endpoint: `https://oauth.platform.intuit.com/oauth2/v1/tokens/bearer`
- API base: `https://quickbooks.api.intuit.com/v3/company/{realmId}` (production) or `https://sandbox-quickbooks.api.intuit.com/v3/company/{realmId}` (sandbox)
- Use env vars: `QBO_CLIENT_ID`, `QBO_CLIENT_SECRET`, `QBO_REDIRECT_URI`, `QBO_ENVIRONMENT` (default "sandbox")
- Token exchange: POST to token endpoint with `grant_type=authorization_code`, `code`, `redirect_uri` in body, Basic auth header with base64(clientId:clientSecret)
- Token refresh: POST with `grant_type=refresh_token`, `refresh_token` in body, same Basic auth
- Chart of accounts: GET `/v3/company/{realmId}/query?query=SELECT * FROM Account WHERE Active = true&minorversion=73` with `Authorization: Bearer {accessToken}`, `Accept: application/json`
- Trial balance: GET `/v3/company/{realmId}/reports/TrialBalance?date_macro=Last Month&minorversion=73` (or `start_date` and `end_date` params for specific period). Parse the Intuit report format (Rows -> Row array with ColData).
- Revoke: POST to `https://developer.api.intuit.com/v2/oauth2/tokens/revoke` with `token={refreshToken}`

For `fetchChartOfAccounts`: Parse QBO Account query response. Map QBO `AccountType` and `AccountSubType` to the `ChartOfAccountsEntry` interface. QBO `Classification` field maps directly.

For `fetchTrialBalance`: Parse the QBO TrialBalance report response format. The response has `Rows.Row[]` where each row has `ColData[{value}]` array. First column is account name, remaining are debit/credit amounts. Map to `TrialBalanceEntry[]`.

**Token manager in src/lib/accounting/token-manager.ts:**

Three exported functions:
1. `storeTokens(connectionId: string, tokens: OAuthTokens)` — Update `AccountingConnection.oauthCredentials` JSON field and `providerCompanyId`/`providerCompanyName` (from QBO company info). Use `prisma.accountingConnection.update`.
2. `getValidTokens(connectionId: string): Promise<OAuthTokens | null>` — Read `oauthCredentials` from DB. If access token is expired (check `accessTokenExpiresAt`), call `refreshAccessToken` and store new tokens. If refresh token is also expired (~100 day cycle), return null and set `syncStatus: ERROR` with `lastSyncError: "OAuth refresh token expired. Please reconnect."`.
3. `refreshTokensIfNeeded(connectionId: string): Promise<OAuthTokens | null>` — Wrapper that gets tokens, refreshes if needed, returns valid tokens or null.

Token expiry: QBO access tokens last ~1 hour, refresh tokens ~100 days. Store `accessTokenExpiresAt` and `refreshTokenExpiresAt` in the JSON.

**Connect endpoint: src/app/api/integrations/qbo/connect/route.ts:**

GET handler. Accepts `?entityId=xxx` query param.
1. Authenticate user with `getAuthUser()`, verify GP_ADMIN role.
2. Verify entity exists and belongs to user's firm.
3. Generate a random `state` parameter (crypto.randomUUID) — encode `{entityId, state}` as base64 to pass through OAuth.
4. Store the state in a cookie (httpOnly, 10 minute expiry) for CSRF protection.
5. Build Intuit authorization URL with: `client_id`, `redirect_uri`, `response_type=code`, `scope=com.intuit.quickbooks.accounting`, `state={encoded}`.
6. Return `NextResponse.redirect(authUrl)`.

**Callback endpoint: src/app/api/integrations/qbo/callback/route.ts:**

GET handler (already whitelisted in middleware).
1. Extract `code`, `realmId`, `state` from query params.
2. Decode state to get entityId. Verify against cookie state for CSRF.
3. Call `qboProvider.exchangeCodeForTokens(code, realmId)`.
4. Call `storeTokens(connectionId, tokens)`.
5. Upsert `AccountingConnection`: set `provider: QBO`, `providerCompanyId: realmId`, `syncStatus: CONNECTED`, `oauthCredentials: tokens`.
6. Fetch QBO company name via GET `/v3/company/{realmId}/companyinfo/{realmId}` and store as `providerCompanyName`.
7. Delete CSRF cookie.
8. Redirect to `/accounting` with success query param (e.g., `?connected=entityId`).

**Disconnect endpoint: src/app/api/integrations/qbo/disconnect/route.ts:**

POST handler. Accepts `{ entityId }` body.
1. Authenticate user, verify GP_ADMIN.
2. Load AccountingConnection for entity.
3. If tokens exist, call `qboProvider.revokeTokens(tokens)` (fire-and-forget — don't fail disconnect if revoke fails).
4. Update connection: `syncStatus: DISCONNECTED`, `oauthCredentials: null`, keep `chartOfAccountsMapped` and `accountMappings` intact (per discretion: keep previously pulled data on disconnect).
5. Return 200.
