<task type="auto">
  <name>Task 1: Audit and fix Zod validation on all mutation API routes</name>
  <files>src/app/api/**/*.ts</files>
  <action>
1. Identify all routes using raw `req.json()` without parseBody:
   ```bash
   grep -rn "req\.json()" src/app/api/ --include="*.ts" | grep -v "parseBody" | grep -v "__tests__"
   ```
   This will show all routes that bypass Zod validation.

2. For EACH route found:
   a. Read the route file to understand the expected request body shape
   b. Check if a Zod schema already exists in `src/lib/schemas.ts` for this resource
   c. If schema exists: Replace `const body = await req.json()` with `const { data, error } = await parseBody(req, ExistingSchema); if (error) return error;`
   d. If no schema exists: Create the Zod schema in `src/lib/schemas.ts`, then use it with parseBody

3. Schema creation guidelines:
   - Use existing Prisma model field types to derive Zod shapes
   - Use `z.string().min(1)` for required string fields
   - Use `z.string().optional()` for optional fields
   - Use `z.nativeEnum(PrismaEnum)` for enum fields (import from `@prisma/client`)
   - Use schemas from `src/lib/json-schemas.ts` (Plan 01) for JSON blob fields
   - For PATCH routes, make all fields optional: `Schema.partial()`
   - Follow existing schema naming convention: `CreateXxxSchema`, `UpdateXxxSchema`

4. Routes to prioritize (from RESEARCH.md — these are the known gaps):
   - Any route under `src/app/api/` that uses `const body = await req.json()` directly
   - Pay special attention to routes handling financial data (capital calls, distributions)
   - Pay special attention to routes handling document categories (`category: category as any` — remove the `as any` cast per CONCERNS.md)

5. Do NOT change:
   - Routes that already use parseBody correctly
   - FormData routes (file uploads use formData, not JSON)
   - GET routes (no request body to validate)
   - DELETE routes (typically no body, or use URL params)

6. After all fixes, verify:
   - `grep -rn "req\.json()" src/app/api/ --include="*.ts" | grep -v "parseBody" | grep -v "__tests__" | grep -v "formData"` should return only FormData routes or zero results
   - `grep -rn "as any" src/app/api/ --include="*.ts"` — reduce count (some may remain for genuinely dynamic content like AI responses)
  </action>
