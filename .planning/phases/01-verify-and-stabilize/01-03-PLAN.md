    - GET returns distributions with entity and line item detail
    - Check if distribution line items (per-investor breakdown) can be created
    - Document: Can a user create a distribution and allocate to LPs?

    **Waterfall Calculation API (VERIFY-04)**
    Read `src/app/api/waterfall-templates/[id]/calculate/route.ts`. Verify:
    - POST accepts entityId and distributableAmount
    - Reads template tiers, entity commitments, prior distributions, first call date
    - Calls computeWaterfall() correctly
    - Stores result in WaterfallCalculation table
    - Returns detailed tier breakdown
    - Document: Does this endpoint work if there are no commitments? No capital calls?

    **Capital Account Compute API (VERIFY-04)**
    Read `src/app/api/investors/[id]/capital-account/compute/route.ts`. Verify:
    - POST accepts entityId, periodStart, periodEnd
    - Reads actual ledger data (capital call line items, distribution line items, income events, NAV, fees)
    - Calls computeCapitalAccount() with correct parameters
    - Upserts result into CapitalAccount table
    - Check: Are the Prisma model names correct? (capitalCallLineItem, distributionLineItem, etc.)
    - Document: Does this produce correct results given actual data?

    **NAV Computation API (VERIFY-04)**
    Read `src/app/api/nav/[entityId]/route.ts`. Verify:
    - GET returns cost basis NAV and economic NAV
    - Reads entity allocations, asset valuations, fee calculations
    - Check: Does it handle entities with no assets?
    - Note: Uses "proxy" values (5% cash, 0.5% other assets, 2% liabilities) — document this as a known approximation

    **Slack IC Voting (VERIFY-02)**
    Read `src/lib/slack.ts` and `src/app/api/slack/interactions/route.ts`. Verify:
    - postICReviewToSlack: Creates Block Kit message with Approve/Reject/View buttons
    - verifySlackSignature: HMAC-SHA256 verification with timing-safe compare and 5-minute window
    - Interaction handler: Parses URL-encoded payload, maps Slack user to Atlas user, records vote, prevents duplicates
    - Graceful degradation: Returns null when SLACK_BOT_TOKEN not configured (no crashes)
    - Check: Does User model have a `slackUserId` field? (Required for user mapping)
    - Check: Does ICProcess model have `slackMessageId` and `slackChannel` fields?
    - Document: What setup is needed to test with a real Slack workspace? (App creation, bot token, signing secret, channel ID, user mapping)
    - NOTE: We cannot test Slack with a real workspace in this plan (requires human setup). Document exactly what's needed so it can be tested later.

    For each feature, categorize:
    - WORKS: Code is correct, API endpoints functional
    - PARTIAL: Code exists but incomplete (e.g., no line item creation)
    - BROKEN: Bug found (describe and fix if possible)
    - UNTESTABLE: Requires external setup (e.g., Slack workspace)
