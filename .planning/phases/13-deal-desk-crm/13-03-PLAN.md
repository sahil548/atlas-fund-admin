**API endpoint (src/app/api/deals/bulk/route.ts):**
1. Create new POST route handler.
2. Use `parseBody(req, BulkDealActionSchema)` for validation.
3. Verify all deal IDs belong to the given firmId (security: prevent cross-tenant manipulation).
4. Implement three action handlers:

**Kill action:**
- Update all deals: `stage = "DEAD"`, `killReason = data.killReason`, `killedAt = new Date()`.
- Use `prisma.deal.updateMany({ where: { id: { in: dealIds }, firmId }, data: { stage: "DEAD", killReason, killedAt: new Date() } })`.
- Log a DealActivity for each deal (use a loop with `prisma.dealActivity.create` — not updateMany since each needs unique dealId). Activity type should be "STAGE_CHANGE" with metadata `{ previousStage, newStage: "DEAD", bulk: true }`.

**Assign action:**
- Validate `assignLeadId` exists and belongs to the firm.
- Update `dealLeadId` on all selected deals: `prisma.deal.updateMany({ where: { id: { in: dealIds }, firmId }, data: { dealLeadId: data.assignLeadId } })`.
- No DealActivity needed for assignment changes (optional — at Claude's discretion).

**Advance action:**
- Fetch all selected deals to determine their current stages.
- Validate ALL deals are in the SAME stage. If not, return 400: `{ error: "All selected deals must be in the same stage to advance" }`.
- Determine next stage: SCREENING -> DUE_DILIGENCE, DUE_DILIGENCE -> IC_REVIEW. If current stage is IC_REVIEW or later, return 400: `{ error: "Cannot bulk advance deals past IC Review — IC decisions required" }`.
- Update all deals to next stage.
- Log DealActivity for each deal with STAGE_CHANGE type.

5. Return `{ updated: number, action: string }` with 200 status.
6. Use `logAudit` for the bulk action (fire-and-forget).
