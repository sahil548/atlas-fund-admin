**Part A: Wire management panels into overview tab**

In `src/components/features/assets/asset-overview-tab.tsx` (created in Plan 03), add conditional rendering of the type-specific panel in the main content area (after key metrics, before entity allocations):

```typescript
import { REManagementPanel } from "./asset-active-management/re-management-panel";
import { FundLPPanel } from "./asset-active-management/fund-lp-panel";
import { CreditManagementPanel } from "./asset-active-management/credit-management-panel";
import { EquityManagementPanel } from "./asset-active-management/equity-management-panel";

// In render, after key metrics:
{asset.assetClass === "REAL_ESTATE" && <REManagementPanel asset={asset} />}
{asset.assetClass === "CREDIT" && <CreditManagementPanel asset={asset} />}
{asset.participationStructure === "LP_POSITION" && <FundLPPanel asset={asset} />}
{(asset.assetClass === "EQUITY" || asset.assetClass === "VENTURE") && <EquityManagementPanel asset={asset} />}
```

Note: LP position check uses `participationStructure` (not assetClass) because fund LP positions can be any asset class but with LP_POSITION participation.

**Part B: Mark Reviewed button + auto-advance nextReview**

In the sidebar section of `asset-overview-tab.tsx`, add a "Mark Reviewed" button below the Next Review date display:
- Button only visible when asset.status === "ACTIVE"
- On click: PATCH `/api/assets/${asset.id}` with `{ nextReview: computedNextReview, reviewedAt: new Date().toISOString() }`
- Compute next review date based on `asset.reviewFrequency`:
  - "quarterly" → 3 months from now
  - "semi_annual" → 6 months from now
  - "annual" → 12 months from now
  - If no reviewFrequency set → 3 months default
- After success: mutate asset data, show toast "Review logged — next review scheduled for {date}"

Update `src/app/api/assets/[id]/route.ts` PUT handler to accept `nextReview` field in the update data (add it to the Zod schema if not already accepted). The existing PUT already handles asset updates — just ensure `nextReview` is included in the allowed fields.
