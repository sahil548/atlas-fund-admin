**Part A: Inline task quick-add component**

Create `src/components/features/tasks/inline-task-add.tsx` — "use client" component:
- Props: `{ contextType: "deal" | "asset" | "entity", contextId: string, contextLabel: string, dealId?: string, assetId?: string, entityId?: string, onTaskCreated?: () => void }`
- State: `showForm` (toggle), `title` (string), `assigneeId` (optional), `dueDate` (optional), `submitting` (boolean)
- Collapsed state: Just a "+ Add Task" button (`text-xs text-indigo-600 hover:underline`)
- Expanded state: Compact inline form (not a modal):
  ```
  [Title input...] [Assignee dropdown] [Due date] [Add button] [Cancel]
  ```
  - Title input: auto-focused, Enter to submit, Escape to cancel
  - Assignee: small dropdown fetching from `/api/users?firmId={firmId}` (optional field)
  - Due date: date input (optional field)
  - Add button: submits
  - Cancel: collapses form
- On submit: POST to `/api/tasks` with:
  ```json
  {
    "title": title,
    "status": "TODO",
    "priority": "MEDIUM",
    "assigneeId": assigneeId || null,
    "dueDate": dueDate || null,
    "contextType": contextType,
    "contextId": contextId,
    "dealId": dealId || null,
    "assetId": assetId || null,
    "entityId": entityId || null
  }
  ```
- CRITICAL: Set BOTH the FK field (dealId/assetId/entityId) AND contextType + contextId — per research anti-pattern #3
- On success: toast "Task created", reset form, collapse, call `onTaskCreated()` to refresh
- Error handling: `const msg = typeof data.error === "string" ? data.error : "Failed to create task"; toast.error(msg);`
