**Part A: Monitoring API endpoint**

Create `src/app/api/assets/monitoring/route.ts` — GET endpoint:
- Accept `firmId` query param (required — return 400 if missing)
- Query in parallel using `Promise.all`:
  a. Covenant breaches: `prisma.covenant.findMany({ where: { currentStatus: { in: ["BREACHED", "BREACH"] }, agreement: { asset: { entityAllocations: { some: { entity: { firmId } } } } } }, include: { agreement: { include: { asset: { select: { id: true, name: true } } } } } })`
  b. Leases expiring within 180 days: `prisma.lease.findMany({ where: { leaseEndDate: { gte: now, lte: day180 }, asset: { entityAllocations: { some: { entity: { firmId } } }, status: "ACTIVE" } }, include: { asset: { select: { id: true, name: true } } } })`
  c. Credit agreements with maturity within 180 days: similar pattern with `maturityDate` filter
  d. Overdue reviews: `prisma.asset.findMany({ where: { nextReview: { lt: now }, status: "ACTIVE", entityAllocations: { some: { entity: { firmId } } } }, select: { id: true, name: true, nextReview: true } })`
- Return JSON: `{ covenantBreaches, leaseExpirations, loanMaturities, overdueReviews, totalAlerts }`
- Use `addDays` helper (inline): `const addDays = (d: Date, n: number) => new Date(d.getTime() + n * 86400000);`
- CRITICAL: All queries must filter through `entityAllocations.some.entity.firmId` — the chain is 4 joins deep per research pitfall #8.
