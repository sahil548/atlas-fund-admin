         - irr: weighted IRR across entity's assets (weighted by fair value)
         - tvpi: entity-level TVPI from commitments and distributions

         - dryPowder: totalCommitted - capitalDeployed
         - topAssets: top 3 assets by fair value with name, fairValue, moic
         - perAssetBreakdown: all assets with costBasis, fairValue, unrealizedGain (for expanded view)
       - Query pattern: entities with includes for assetAllocations -> asset, commitments, capitalCalls, distributions
       - Use computeMetrics from src/lib/computations/metrics.ts for TVPI calculation
       - Scope by firmId from query params
    2. Create /api/dashboard/portfolio-aggregates/route.ts:
       - GET: Aggregate across all entities:
         - assetAllocation: reuse existing /api/dashboard/asset-allocation logic (by asset class)
         - topPerformers: top 5 assets ranked by IRR (with name, irr, moic, fairValue, entityName)
         - bottomPerformers: bottom 5 assets ranked by IRR (same shape)
         - capitalDeployment: { totalCommitted, totalCalled, totalDeployed, totalDryPowder } per entity + aggregate
         - recentActivity: last 10 actions across the firm (capital calls, distributions, valuations, deal closes) — query DealActivity + CapitalCall + DistributionEvent, merge and sort by date
       - Scope by firmId

         - aggregateMetrics: { tvpi, dpi, rvpi, irr } across all entities
       - Use computeMetrics for each investor-entity pair
       - Sort by totalCommitted descending
       - Scope by firmId
