  <name>Task 1: Create personal AI config API endpoint</name>
  <files>src/app/api/users/[id]/ai-config/route.ts</files>
  <action>
Create a new API route at `src/app/api/users/[id]/ai-config/route.ts` with GET and PUT handlers.

**GET handler:**
1. Await and extract `{ id }` from params (Next.js 16 -- params is a Promise)
2. Call `getAuthUser()` to verify authentication
3. Fetch the user: `prisma.user.findUnique({ where: { id }, select: { aiEnabled: true, personalAiConfig: true, firmId: true } })`
4. If user not found, return 404
5. Call `getUserAIConfig(id, user.firmId)` to get the resolved config with source indicator
6. Return JSON with: `{ aiEnabled, provider, model, hasPersonalKey: boolean, source }` -- NEVER return the actual API key to the client. Return whether a key exists and the source.

**PUT handler:**
1. Await and extract `{ id }` from params
2. Call `getAuthUser()` to verify authentication
3. Parse request body -- accept: `{ provider?: string, model?: string, apiKey?: string | null }`
4. Validate: provider must be "openai" or "anthropic" if provided. Model must be a non-empty string if provided.
5. Build the personalAiConfig JSON object:
   - If `apiKey` is provided and non-null, encrypt it: `const { encrypted, iv, tag } = encryptApiKey(apiKey)`
   - Store as: `{ provider, model, apiKey: encrypted, apiKeyIV: iv, apiKeyTag: tag }`
   - If `apiKey` is explicitly null, clear the personal key: set personalAiConfig to `{ provider, model }` (no key fields)
   - If `apiKey` is not provided, preserve existing encryption, only update provider/model
6. Update the user: `prisma.user.update({ where: { id }, data: { personalAiConfig: configJson } })`
7. Return the updated config (same shape as GET -- no raw key in response)

Import `encryptApiKey`, `getUserAIConfig` from `@/lib/ai-config`. Import `prisma` from `@/lib/prisma`. Import `getAuthUser` from `@/lib/auth`. Use `parseBody` from `@/lib/api-helpers` with a Zod schema for the PUT body.

Follow the Next.js 16 dynamic route params pattern: `{ params }: { params: Promise<{ id: string }> }` and `const { id } = await params;`
