  **A. In-app voting endpoint:**

  File: src/app/api/deals/[id]/ic/vote/route.ts

  POST:
  - Accept { vote: "APPROVE" | "REJECT" | "SEND_BACK", conditions?: string, userId: string }
  - Fetch the deal's ICProcess. If none exists, return 404.
  - Check user hasn't already voted (query ICVoteRecord for this userId + icProcessId). If duplicate, return 409.
  - Create ICVoteRecord with: vote, notes: conditions || null, conditions: conditions || null
  - Check if quorum reached:
    - Get entity linked to deal (deal.entityId or DealEntity records)
    - Get the entity's DecisionStructure (if any)
    - Count votes. If total approve >= approvalThreshold, structure allows advancing.
    - Do NOT auto-advance; just update ICProcess status to reflect vote count
  - Return 201 with the vote record

  **B. Send Back flow in deal-stage-engine.ts:**

  Add sendBackToDueDiligence(dealId: string, reason: string):
  - Set deal.stage = DUE_DILIGENCE
  - Log DealActivity with type "DEAL_SENT_BACK" and metadata: { reason }
  - Update ICProcess status to "sent_back"
  - Return updated deal
