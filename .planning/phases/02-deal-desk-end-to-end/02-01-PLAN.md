  **Part A: InlineEditField reliability (DEAL-05)**

  Fix edge cases in inline-edit-field.tsx:
  1. **Prevent double-save**: Add a `savingRef` (useRef) to guard against concurrent save calls. If already saving, skip.
  2. **Blur after Enter**: When Enter triggers save(), the blur event fires immediately after. The blur handler should check if we just saved (use a ref flag that resets after a short timeout) and skip the duplicate save.
  3. **Error feedback**: If the PUT fails (non-2xx), show a toast error and keep the edit mode open so the user doesn't lose their input. Import useToast.
  4. **Textarea Enter**: For textarea type, Enter should NOT save (it inserts a newline). Only blur saves for textarea. Ctrl+Enter can optionally save.

  **Part B: Kill deal with reason + revive**
