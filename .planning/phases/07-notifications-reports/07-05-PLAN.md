  <action>
    1. Add to Prisma schema if not present:
       - Add `DocuSignConnection` model for storing OAuth tokens at firm level:
         ```
         model DocuSignConnection {
           id            String   @id @default(cuid())
           firmId        String   @unique
           accessToken   String
           refreshToken  String
           tokenExpiry   DateTime
           accountId     String   // DocuSign account ID
           baseUri       String   // API base URI from /oauth/userinfo
           createdAt     DateTime @default(now())
           updatedAt     DateTime @updatedAt
         }
         ```
       - Run `npx prisma generate` after schema changes

    2. Create `src/lib/docusign.ts`:
       - Use raw fetch (no SDK — same approach as QBO OAuth in Phase 5, per established pattern)
       - Export class `DocuSignClient`:
         - constructor(connection: DocuSignConnection)
         - `static getAuthUrl(firmId: string)`: generates OAuth authorization URL with state param
         - `static exchangeCode(code: string)`: exchanges auth code for access+refresh tokens
         - `async refreshTokenIfNeeded()`: checks tokenExpiry, refreshes if < 5 min remaining
         - `async createEnvelope({ documentUrl, documentName, signers, subject })`: creates envelope via DocuSign REST API, returns envelopeId
         - `async getEnvelopeStatus(envelopeId: string)`: returns current status
         - `async downloadSignedDocument(envelopeId: string)`: downloads completed document as buffer

       - OAuth URLs:
         - Auth: `${DOCUSIGN_OAUTH_BASE}/oauth/auth?response_type=code&scope=signature&client_id=${INTEGRATION_KEY}&redirect_uri=${CALLBACK_URL}&state=${STATE}`
         - Token: `${DOCUSIGN_OAUTH_BASE}/oauth/token` (POST with grant_type=authorization_code)
         - Refresh: same endpoint with grant_type=refresh_token

       - Envelope creation: POST `${baseUri}/v2.1/accounts/${accountId}/envelopes` with:
         - documents array (base64 encoded), recipients.signers array, status: "sent"

    3. Create `src/app/api/docusign/connect/route.ts`:
       - GET: generates OAuth URL, sets state in httpOnly cookie (same CSRF pattern as QBO), redirects to DocuSign
       - firmId from query params

    4. Create `src/app/api/docusign/callback/route.ts`:
       - GET: exchanges code for tokens, stores in DocuSignConnection, redirects back to /settings

    5. Create `src/app/api/docusign/disconnect/route.ts`:
       - POST: deletes DocuSignConnection for the firm

    6. Create `src/app/api/docusign/webhook/route.ts`:
       - POST: receives DocuSign Connect webhook events (no auth header — DocuSign uses HMAC or custom header)
       - Parse event body for envelopeId and status
       - Map DocuSign status to ESignatureStatus:
         - "sent" -> SENT, "delivered" -> VIEWED, "completed" -> COMPLETED, "declined" -> DECLINED
       - Update ESignaturePackage by externalId: set status, completedAt if completed
       - If COMPLETED: download signed document, upload to Vercel Blob, update Document record with signed version URL
