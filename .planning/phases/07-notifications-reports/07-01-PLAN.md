    2. Create `src/lib/email.ts`:
       - Export `sendEmail({ to, subject, html, from? })` — uses Resend SDK
       - `from` defaults to `process.env.EMAIL_FROM_ADDRESS || "notices@atlas.com"`
       - Fire-and-forget pattern: caller awaits but errors are caught and logged, never thrown
       - If RESEND_API_KEY is missing, log warning and skip (graceful degradation for dev)

    3. Create `src/lib/email-templates.ts`:
       - Export `capitalCallEmailHtml({ entityName, callNumber, amount, dueDate, purpose, wiringInstructions })` — returns clean HTML string
       - Export `distributionPaidEmailHtml({ entityName, distributionDate, grossAmount, netToLPs })` — returns clean HTML string
       - Export `reportAvailableEmailHtml({ entityName, reportType, period, portalUrl })` — returns clean HTML string
       - Export `k1AvailableEmailHtml({ entityName, taxYear, portalUrl })` — returns clean HTML string
       - Style: professional, mobile-responsive, simple card layout per user decision
       - White-label: entity/fund name as sender identity, NO Atlas branding per user decision
       - Capital call emails include FULL details (amount, due date, wiring instructions) per user decision — LP doesn't need to log in

    4. Create `src/lib/sms.ts`:
       - Export `sendSMS({ to, body })` — uses Twilio REST API via fetch (POST to https://api.twilio.com/2010-04-01/Accounts/{SID}/Messages.json with Basic auth)
       - `from` is `process.env.TWILIO_PHONE_NUMBER`
       - Fire-and-forget: errors caught and logged
       - If TWILIO env vars missing, log warning and skip

    5. Create `src/lib/notification-delivery.ts`:
       - Export `deliverNotification({ investorId, type, subject, body, emailHtml, smsBody })`:
         a. Fetch InvestorNotificationPreference for investorId
         b. Check notificationTypes — if the type category is disabled, skip external delivery (still create in-app)
         c. Based on preferredChannel: EMAIL -> sendEmail, SMS -> sendSMS, PORTAL_ONLY -> in-app only
         d. Always create in-app notification via createNotification (pass investorId to create notification for their linked userId)
         e. Update Notification record: set sentAt, status="sent", channel

       - Export `notifyInvestorsOnCapitalCall(capitalCallId: string)`:
         a. Fetch CapitalCall with lineItems -> investor (include InvestorNotificationPreference, and investor.user for userId)
         b. Fetch entity name for the capital call
         c. For each line item investor: call deliverNotification with type "capitalActivity", capitalCallEmailHtml, short SMS body
         d. Also call notifyGPTeam for in-app GP notification

       - Export `notifyInvestorsOnDistribution(distributionId: string)`:
         a. Same pattern as capital call but for distribution PAID
         b. Fetch DistributionEvent with lineItems -> investor
         c. For each investor: deliverNotification with distributionPaidEmailHtml

       - Export `notifyInvestorsOnReportAvailable({ investorIds, entityName, reportType, period })`:
         a. For each investor: deliverNotification with reportAvailableEmailHtml
         b. Type category: "reports"

       - Export `notifyInvestorsOnK1Available({ investorId, entityName, taxYear })`:
         a. deliverNotification with k1AvailableEmailHtml
         b. Type category: "reports"

    6. Extend `src/lib/notifications.ts`:
       - Add `DISTRIBUTION` and `REPORT` to NotificationType union (also add to Prisma enum in schema)
       - Update createNotification to accept optional `investorId` parameter (for LP notifications)

    7. Update `prisma/schema.prisma`:
       - Add `DISTRIBUTION` and `REPORT` to NotificationType enum
       - Run `npx prisma generate` (no db push needed for enum addition — will be handled at build)
