       - POST handler accepting multipart/form-data (FormData with multiple files + entityId + taxYear)
       - Do NOT set Content-Type header (browser sets multipart boundary automatically)
       - Parse files from FormData
       - Fetch all investors for the entity (via commitments): `prisma.investor.findMany({ where: { commitments: { some: { entityId } } } })`
       - For each uploaded file:
         a. Extract investor name from filename pattern: "K1_InvestorName_2025.pdf" -> "InvestorName"
         b. Normalize: replace underscores/hyphens with spaces, trim, lowercase for matching
         c. Match against investor names (case-insensitive, fuzzy: check if investor name contains extracted name or vice versa)
         d. Upload to Vercel Blob: `put(`k1/${entityId}/${taxYear}/${file.name}`, buffer, { access: "public", contentType: "application/pdf" })`
         e. Create Document record: `{ name: file.name, category: "TAX", fileUrl: blobUrl, fileSize, mimeType: "application/pdf", entityId, investorId: matchedInvestor?.id }`
         f. If investor matched: fire-and-forget `notifyInvestorsOnK1Available({ investorId: matchedInvestor.id, entityName, taxYear })`
       - Return { uploaded: number, matched: number, unmatched: string[] (filenames that couldn't match) }
       - If some files can't match, they're still uploaded as Documents (just without investorId) — GP can manually assign later

    2. Create `src/app/api/k1/route.ts`:
       - GET handler: list K-1 documents (Documents with category TAX)
       - Filter by firmId (via entity), optional entityId and taxYear query params
       - Include entity name and investor name in response
       - Order by createdAt desc

    3. Add schema to `src/lib/schemas.ts` if needed for validation.
