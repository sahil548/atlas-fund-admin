LP Portfolio API — investedCapital derivation pattern (Prisma aggregate, NOT HTTP):
```typescript
// Do NOT call /api/lp/[investorId]/dashboard from inside this route —
// HTTP-from-server is an antipattern (extra roundtrip, fragile, auth-confused).
// Use a direct Prisma aggregate on capitalCallLineItem filtered by status FUNDED.
//
// For each asset in the portfolio response:
// 1. Determine the asset's parent entity (asset.entityId, or via assetEntityAllocation
//    if an asset can live under multiple entities — in that case pick the primary
//    allocation or sum across, matching the existing proRata convention).
// 2. Aggregate FUNDED capital calls for this investor + entity.
// 3. Multiply the aggregated totalCalled by investorPct to get the LP's share.

const { _sum } = await prisma.capitalCallLineItem.aggregate({
  _sum: { amount: true },
  where: {
    investorId,
    capitalCall: { status: "FUNDED" },
    entityId, // from the asset's parent entity
  },
});
const totalCalled = _sum.amount ?? 0;
const investedCapital = totalCalled * investorPct;
```

Confirm exact Prisma model and field names from `prisma/schema.prisma` when editing.
The model name `capitalCallLineItem` and the relation traversal `capitalCall: { status: "FUNDED" }` follow the existing v2.1 conventions used in the capital account route (per Plan 22-03's research); use the same shapes here to keep behavior consistent.

**API change — `src/app/api/lp/[investorId]/portfolio/route.ts`:**

Per RESEARCH.md, the existing response includes `data.assets[].proRata` (fair-value-based) and `data.assets[].investorPct`. It does NOT include invested capital. Derive invested capital per holding via a server-side Prisma aggregate on `capitalCallLineItem`.

**Do NOT make an HTTP call from inside this server route.** HTTP-from-server (e.g. calling `/api/lp/[investorId]/dashboard`) adds an unnecessary roundtrip, complicates auth plumbing, and is brittle when request context is not forwarded. Use a Prisma aggregate instead.

**Steps:**

1. **Confirm model/field names.** Open `prisma/schema.prisma` and confirm:
   - Model name `CapitalCallLineItem` (Prisma client accessor: `capitalCallLineItem`)
   - Fields: `investorId`, `entityId`, `amount`, and the relation `capitalCall` with a `status` enum that includes `FUNDED`
   - If any field or model name differs (e.g. `CallLineItem` or `drawLineItem`), use the actual name from schema and note the substitution in VERIFICATION.md.

2. **For each asset in the portfolio response, derive invested capital via aggregate:**

   ```typescript
   // Inside the GET handler, after loading the asset list:
   const assetsWithInvested = await Promise.all(
     assets.map(async (asset) => {
       // Determine parent entity for this asset. If asset.entityId is direct,
       // use it; if assets can span multiple entities via assetEntityAllocation,
       // pick the primary allocation (or sum totalCalled across all entities
       // weighted by allocationPercent — match whatever convention proRata uses
       // so the two columns are internally consistent).
       const entityId = asset.entityId;

       const { _sum } = await prisma.capitalCallLineItem.aggregate({
         _sum: { amount: true },
         where: {
           investorId,
           entityId,
           capitalCall: { status: "FUNDED" },
         },
       });
       const totalCalled = _sum.amount ?? 0;
       const investedCapital = totalCalled * asset.investorPct;

       return { ...asset, investedCapital };
     }),
   );
   ```

3. **N+1 consideration.** If the `assets` array is >50 items, the per-asset aggregate becomes an N+1. Optimize by aggregating once per entity (dedupe entityIds first), then mapping back:

   ```typescript
   const uniqueEntityIds = Array.from(new Set(assets.map((a) => a.entityId)));
   const perEntityTotals = new Map<string, number>();
   await Promise.all(
     uniqueEntityIds.map(async (entityId) => {
       const { _sum } = await prisma.capitalCallLineItem.aggregate({
         _sum: { amount: true },
         where: { investorId, entityId, capitalCall: { status: "FUNDED" } },
       });
       perEntityTotals.set(entityId, _sum.amount ?? 0);
     }),
   );
   const assetsWithInvested = assets.map((asset) => ({
     ...asset,
     investedCapital: (perEntityTotals.get(asset.entityId) ?? 0) * asset.investorPct,
   }));
   ```

4. Return `assetsWithInvested` instead of `assets` in the response.
