# Plan 22-11: Asset Review Schedule + Ownership Tracking

<objective>
User flagged two more gaps after 22-10 shipped: "how do I set the asset review schedule" — and it turned out `reviewFrequency` (quarterly / semi-annual / annual) and `nextReview` weren't exposed in the Add/Edit forms despite existing on the Asset model. Same gap pattern for `ownershipPercent`, `shareCount`, and `hasBoardSeat`.

User decision on 2026-04-17: keep Phase 22 open until this lands, same way 22-10 was bundled.

Five new scalar fields lifted into Add + Edit forms:
- `nextReview` (DateTime) — when the next asset review is due
- `reviewFrequency` (enum: quarterly / semi_annual / annual) — auto-advance cadence used by the existing "Mark Reviewed" button
- `ownershipPercent` (Float) — % of underlying asset owned (e.g., 18.4%)
- `shareCount` (Float) — shares/units held
- `hasBoardSeat` (Boolean) — directorship toggle
</objective>

<must_haves>
<truths>
- `UpdateAssetSchema` accepts the five new fields; `CreateAssetSchema` accepts them at creation time
- `POST /api/assets` persists the five fields when supplied in the request body
- `PUT /api/assets/[id]` accepts the five fields via the existing `...rest` spread (no explicit destructure needed)
- Edit Asset modal renders a "Review & Ownership" fieldset containing Next Review (date), Review Frequency (select), Ownership %, Share Count, and a Has-board-seat checkbox
- Add Asset modal renders the same fieldset with defaults (no Next Review / no Frequency / empty values / checkbox unchecked)
- Editing and saving persists all five fields; re-opening Edit pre-fills them from the fetched asset
- `npm run build` passes with zero TypeScript errors
</truths>
</must_haves>
