**Create `src/app/api/capital-calls/[id]/route.ts`:**
- GET: Fetch capital call by ID with entity, lineItems (include investor), and summary stats
- PATCH: Status transition. Accept `UpdateCapitalCallSchema`. Validate transitions:
  - DRAFT -> ISSUED (allowed)
  - ISSUED -> PARTIALLY_FUNDED (auto, not manual — computed from line items)
  - ISSUED/PARTIALLY_FUNDED -> FUNDED (auto, not manual)
  - Any non-FUNDED -> OVERDUE (allowed manually, or detected on page load when dueDate < now)
  - No backward transitions allowed (return 400)
  When status is changed to ISSUED, check if dueDate is in the past and warn (but allow).

**Create `src/app/api/capital-calls/[id]/line-items/route.ts`:**
- GET: List all line items for a capital call (include investor name, commitment info)
- POST: Create line item. Accept `CreateCapitalCallLineItemSchema`. Validate:
  - Investor has a commitment to the capital call's entity
  - Warn (header, not block) if line item amount would exceed unfunded commitment (per user decision: "Warn but allow")
  - Check for duplicate investor (same investor already has a line item for this call)

**Create `src/app/api/capital-calls/[id]/line-items/[lineItemId]/route.ts`:**
- PATCH: Update line item. Accept `UpdateCapitalCallLineItemSchema`.
  - When status changes to "Funded":
    1. Set paidDate to provided date or now
    2. Call `updateCommitmentCalledAmount(investorId, entityId)`
    3. Call `recomputeCapitalAccountForInvestor(investorId, entityId)`
    4. Call `updateCapitalCallStatus(capitalCallId)` to auto-update parent status/fundedPercent

**Create `src/app/api/distributions/[id]/route.ts`:**
- GET: Fetch distribution by ID with entity, lineItems (include investor), and summary stats
- PATCH: Status transition. Accept `UpdateDistributionSchema`. Validate transitions:
  - DRAFT -> APPROVED (allowed)
  - APPROVED -> PAID (allowed)
  - No backward transitions (return 400)
  When transitioning to PAID:
    1. Call `recomputeAllInvestorCapitalAccounts(entityId)` to recompute all investor capital accounts
  Also allow updating fields (grossAmount, distributionType, memo, decomposition fields) when status is DRAFT.

**Create `src/app/api/distributions/[id]/line-items/route.ts`:**
- GET: List all line items for a distribution (include investor name)
- POST: Create line item. Accept `CreateDistributionLineItemSchema`.

**Create `src/app/api/distributions/[id]/line-items/[lineItemId]/route.ts`:**
- PATCH: Update line item. Accept `UpdateDistributionLineItemSchema`. Only allow updates when parent distribution status is DRAFT.
