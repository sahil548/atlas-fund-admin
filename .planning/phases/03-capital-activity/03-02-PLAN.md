Update `computeWaterfall` signature to accept optional `config: WaterfallConfig`:
```typescript
export function computeWaterfall(
  tiers: WaterfallTierInput[],
  distributableAmount: number,
  totalContributed: number,
  totalDistributedPrior: number,
  yearsOutstanding: number,
  config?: WaterfallConfig,
): WaterfallResult;
```

Changes to waterfall logic:
1. GP Catch-Up tier: replace hardcoded `gpTargetPct = 0.20` with `config.carryPercent ?? 0.20`
2. Preferred Return tier:
   - If compounding === "COMPOUND": `prefAmount = totalContributed * (Math.pow(1 + hurdleRate/100, yearsOutstanding) - 1)`
   - If compounding === "SIMPLE" (default): keep current logic `totalContributed * (hurdleRate/100) * yearsOutstanding`
   - If `prefReturnOffsetByDistributions`: reduce prefAmount by `totalDistributedPrior`
   - If `incomeCountsTowardPref`: reduce prefAmount by `config.priorIncomeDistributed ?? 0`
3. GP co-invest: when `gpCoInvestPercent` is set, split the LP allocation — GP co-invest portion gets LP treatment (ROC + pref) and then GP also gets carry on the GP side. Add `gpCoInvestAllocation` to result.
4. Add `clawbackLiability` to WaterfallResult: calculate as max(0, totalGPReceived_including_prior - totalGPEntitled). Where totalGPEntitled = total profits * carryPercent. This is a display-only field.
5. Add `perInvestorBreakdown` to WaterfallResult (optional): array of { investorId, proRataShare, lpAllocation, gpCarryAllocation? }. The API endpoint will compute this from commitment data and pass investor shares.

Keep backward compatibility — existing callers without config get the same results (defaulting to 20% carry, simple pref, etc.).
